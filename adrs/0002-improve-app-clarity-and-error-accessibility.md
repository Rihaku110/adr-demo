# ADR: Appの文言明確化とエラー表示アクセシビリティ改善

- **ID**: ADR-002
- **日付**: 2026-04-28
- **ステータス**: accepted

## 決定
`App.jsx` において、以下の最小改善を行う。
- カウント更新を関数型更新（`setCount((prev) => prev + 1)`）に変更する
- エラー状態を真偽値ではなくメッセージ文字列で管理する
- エラー表示に `role="alert"` と `aria-live="assertive"` を付与する

## 根拠
なぜこの方式を選択したか:
- 関数型更新は、連続クリック時でも最新状態に対して安全に加算できるため
- エラーを文字列で管理すると、ユーザーに何が起きたかを具体的に伝えられるため
- `role="alert"` と `aria-live` により、視覚以外の利用者にもエラー発生を明確に伝えられるため

## 代替案
- 現状維持（最小工数だが、状態更新とエラー通知の明確性が不足する）
- 大規模な状態管理導入（小規模デモには過剰）
- エラーをコンソール出力のみとする（ユーザーへ状況が伝わらない）
