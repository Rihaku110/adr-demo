# ADR: Appのビジュアルデザイン刷新

- **ID**: ADR-003
- **日付**: 2026-04-28
- **ステータス**: accepted

## 決定
`App.css` と `App.jsx` を更新し、Reactデモアプリの見た目をモダンで視認性の高いデザインに刷新する。

- 背景にグラデーション + ぼかし円のアニメーションを追加（ライト/ダーク両対応）
- カードを半透明＋backdrop-filterによるグラスモーフィズム化
- ボタンをグラデーション化し、hover/active/focus状態を強化
- 見出し `h1` とステータス値にグラデーションテキストを適用
- ステータス表示をラベル＋数値の縦積みグリッドに再構成し、押下回数を視覚的に強調
- エラー表示にアイコンとスライドインアニメーションを追加

## 根拠
なぜこの方式を選択したか:
- グラスモーフィズムとグラデーションは、最小限のCSSで「かっこよさ」と階層感を出せる定番手法のため
- ステータス値を大きなグラデーションテキストにすることで、ユーザーが現在値を一目で把握できるため
- `:focus-visible` とコントラスト比の高い配色を維持し、アクセシビリティ（ADR-002の方針）を後退させないため
- アニメーションは控えめにし、`prefers-reduced-motion` を阻害しない範囲に留めるため
- JSXの構造変更は最小限（status-box内のラベル/値分離のみ）に抑え、機能差分を出さないため

## 代替案
- UIライブラリ（MUI, Chakra UIなど）の導入: デモ規模に対して依存追加コストが大きい
- TailwindCSS導入: 設定変更が必要で、現状のCSS構成と二重管理になる
- 現状維持: 「もっと見やすくかっこよく」というユーザー要求を満たせない
