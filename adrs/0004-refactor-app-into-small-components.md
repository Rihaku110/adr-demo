# 0004 App.jsx を小さなコンポーネントに分割して可読性を向上

## 決定
`App.jsx` 内の JSX を `Header` / `ControlButtons` / `StatusItem` / `StatusBox` / `ErrorAlert` の小さな関数コンポーネントに分割し、エラー文言は `ERROR_MESSAGE` 定数として抽出した。`App` 本体はステートとハンドラ定義、構成要素の組み立てに専念する形に整理した。

## 根拠
- 単一の `App` に header / ボタン群 / ステータス表示 / エラー表示が混在しており、JSX の役割境界が読み取りづらかった。
- 役割ごとにコンポーネント化することで、各ブロックの責務が名前から即座にわかり、差分レビューやテストの単位も明確になる。
- エラー文言をモジュール先頭の定数に切り出すことで、文言修正時の変更箇所が一意になり、トグル処理から表示文言の関心事を分離できる。
- 既存の DOM 構造・CSS クラス・アクセシビリティ属性 (`role="alert"`, `aria-live`) は維持しているため、UI 仕様への影響は無い（最小差分の原則を満たす）。

## 代替案
- **そのまま維持**: 規模は小さいが、今後機能追加時に同じ箇所が肥大化しやすいため不採用。
- **別ファイルへコンポーネント分割（`components/` ディレクトリ作成）**: 過剰設計。現状はファイル内分割で十分読みやすく、ファイル数増加によるナビゲーションコストを避けた。
- **カスタムフックへ状態ロジック抽出 (`useCounter` 等)**: ロジックは十分単純で、抽象化の利得より読み手の追跡コストが上回るため見送り。
