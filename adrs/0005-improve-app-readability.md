# 0005 App.jsx の可読性をさらに向上

## 決定
`App.jsx` の構造を変えずに、以下の最小リファクタを行い読みやすさを向上させた。

- セクション区切りコメント (`// ---------- 定数 ----------` / `プレゼンテーショナルコンポーネント` / `App 本体`) を追加し、ファイル内のスクロール時に役割境界が一目で分かるようにした。
- マジックストリング `"light"` / `"dark"` を `THEME_LIGHT` / `THEME_DARK` 定数として抽出。
- 冗長だった `isDark` / `themeLabel` の派生変数を廃止し、`theme` ステートを直接利用する形に統一。
- `handleToggleTheme` を `setTheme((prev) => ...)` の関数型更新に揃え、他のハンドラ (`handleCountUp` / `handleToggleError`) と書き味を一致させた。
- 各ハンドラ間に空行を入れ、視覚的な分離を強化。

## 根拠
- 既に 0004 でコンポーネント分割は済んでいるため、追加の構造変更（ファイル分割や状態ロジックのフック化）は過剰になる。読み手の負担を増やさず「見やすさ」だけを改善するスコープに限定した。
- セクションコメントは IDE のアウトライン代わりとして機能し、ファイル内ナビゲーションコストを下げる。
- テーマ文字列を定数化することで、タイプミスによる比較ミス（`"drak"` 等）を防止し、将来的な値変更時の修正点が一箇所になる。
- `isDark` / `themeLabel` は `theme` と 1:1 対応する派生値で、可読性向上というより読み解きの分岐を増やしていたため削除した。`className` には `theme` をそのまま使えば十分。
- ハンドラの書式統一は差分レビュー時のノイズを減らし、プロジェクトとしての一貫性を担保する。
- DOM 構造・CSS クラス・アクセシビリティ属性は変更していないため、UI/挙動への影響は無い。

## 代替案
- **`components/` ディレクトリへのファイル分割**: 0004 の代替案と同じ理由で見送り。現状の規模ではファイル内分割が最も読みやすい。
- **`useReducer` への置き換え**: 状態が独立した3つのみで関連ロジックも持たないため、reducer 化は抽象化コストの方が大きく不採用。
- **`useCounter` / `useToggle` などのカスタムフック抽出**: 各ロジックが 1〜2 行で完結しており、フック化すると逆に追跡コストが増えるため見送り。
- **何もしない**: ユーザーから明示的に「見やすくして」と依頼されており、最小差分での改善余地があったため不採用。
