# 0006. App の状態管理をカスタムフックに抽出する

- ステータス: 採用
- 日付: 2026-04-28

## 決定

`App.jsx` 内の `useState` と各種ハンドラ（`handleCountUp` / `handleToggleError` / `handleToggleTheme`）を、同ファイル内の新しいカスタムフック `useAppState` に集約する。`App` コンポーネント本体は `useAppState()` の戻り値を分割代入し、JSX の組み立てに専念する。

## 根拠

- これまでの `App` 本体は「状態定義 → ハンドラ定義 → JSX」が一画面に縦並びで混在しており、見通しが悪かった。
- 状態とそれに付随する更新ロジックを `useAppState` に括り出すことで、`App` 本体は「何を表示するか」だけを記述する宣言的な構造になり、上から読んだときの認知負荷が下がる。
- ハンドラ名と Props 名（`onCountUp` など）を揃えることで、`ControlButtons` への受け渡しがそのまま分割代入で完結し、命名のブレもなくなる。
- ファイル分割は行わず最小差分に留めたため、既存のテスト・ビルド設定への影響がない。

## 代替案

1. **状態とハンドラを `App` 直下のまま据え置く**: 変更コストはゼロだが、今回の目的（読みやすさ向上）を達成できないため不採用。
2. **`useAppState` を別ファイル（例: `src/hooks/useAppState.js`）に切り出す**: 再利用やテストはしやすくなるが、現時点で `App` 以外から使われる予定がなく、ファイル数が増える割に得るものが少ないため見送り。将来的に再利用が必要になった段階で切り出す。
3. **`useReducer` でアクション駆動にまとめる**: 状態が増えたときに有効だが、現状は 3 つの独立した状態しかなく、reducer 化はかえって冗長になるため不採用。
