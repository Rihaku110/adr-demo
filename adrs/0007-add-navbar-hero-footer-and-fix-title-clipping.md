# 0007. サイト構造の本格化とヒーロータイトルの表示崩れ修正

- ステータス: 採用
- 日付: 2026-05-04

## 決定

`App.jsx` に `Navbar` / `Hero` / `Footer` の各コンポーネントを追加し、サイト全体を「ナビバー → ヒーロー → カード → フッター」というプロダクトサイト的なレイアウトに再構成する。あわせて、グラデーション文字（`React Demo App`）の上下が見切れていた問題を、以下の CSS 変更で解消する。

- `.app` の `overflow: hidden` を `overflow-x: clip` に変更し、縦方向のクリップを止める
- `.hero-title` に `line-height: 1.2` と `padding: 0.15em 0.05em` を付与し、グラデーション文字（`background-clip: text`）の上端・下端の欠けを防ぐ
- `index.css` の `#root` に固定値で入っていた `width: 1126px` / `border-inline` / `text-align: center` を撤廃し、新レイアウト側（`.container` / `.navbar-inner`）で幅とセンタリングを制御する

## 根拠

- ユーザー報告のスクリーンショットでタイトル `React Demo App` の上端が切れていた。原因は (1) `.app { overflow: hidden }` と (2) `clamp()` で大きくなったフォントに対して `line-height` が暗黙の `normal` のまま padding も無く、グラデーション文字のメトリクスがコンテナをはみ出していたこと。`overflow-x: clip` + 明示的な `line-height` と上下 `padding` で構造的に解決でき、将来フォントサイズを変えても再発しない。
- 「本格的なサイトに見せる」要件に対しては、最小コストで効果が大きい順に固定ナビバー（sticky + backdrop-filter）、エフェメラル（eyebrow ラベル付き）ヒーロー、コピーライト付きフッターを追加。既存のカード／状態表示／エラー表示コンポーネントには手を入れず、外側の枠だけを整えることで差分を局所化した。
- `index.css` の `#root` は固定幅 1126px と中央ボーダーを持っており、新しいフルブレッドのナビバー／フッターと衝突する。固定値を取り除き、レイアウト責務を `App.css` 側のセクション単位コンテナに集約することで、ナビバーのスティッキー挙動と背景グラデーションの全幅表示が両立できる。

## 代替案

1. **`h1` の `font-size` を小さくして見切れを回避**: 一見手早いが、原因（`overflow: hidden` と line-height）が残るためフォントや言語切替で再発する。対症療法のため不採用。
2. **`overflow: hidden` を残し、`.app` の `padding-top` だけ増やす**: ヒーローはマシになるが、将来的に他のグラデーション見出しを追加するたびに調整が必要。根本原因（縦方向クリップ）を残すため不採用。
3. **既存レイアウトのまま色やシャドウだけリッチ化**: 差分は最小だが、「本格的なサイト」に求められる導線（ナビ）と締め（フッター）が無いままなので要件を満たさず不採用。
4. **UI ライブラリ（MUI / Chakra など）を導入**: 見栄えは出るが、依存追加・バンドル肥大・既存スタイルとの衝突が大きい。デモアプリの規模に対して過剰であり不採用。
