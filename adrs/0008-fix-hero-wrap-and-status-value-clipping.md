# 0008. ヒーロー説明文の改行抑止と status-value の見切れ修正

- ステータス: 採用
- 日付: 2026-05-04

## 決定

`App.css` の `.hero-description` と `.status-value` を以下のとおり調整する。

- `.hero-description.max-width` を `580px` → `680px` に拡張し、「Copilotと対話しながらコードを変更し、そのたびにADRを自動生成するデモ用アプリです。」が「アプリです」だけ次行へ落ちるのを防ぐ。
- `.status-value` の `font-size` を `1.5rem` → `1.35rem` に下げ、`line-height: 1.4` と `padding: 0.1em 0.05em` を追加し、`light` のディセンダ（g）が下端で切れないようにする。

## 根拠

- スクリーンショットで `アプリです` が最終行に単独で改行されており、視覚的にバランスが悪かった。`max-width` をわずかに広げるだけで 1 行差を吸収でき、レイアウト崩れも起こらない。
- `light` の見切れは、`background-clip: text` で描画したテキストのディセンダが、行ボックスの底に密着し親の角丸枠と衝突して切れて見えていたのが原因。`line-height` を明示しつつ上下に微小な `padding` を確保することで、フォントサイズに依存せず再発しない構造になる。フォントサイズを少し下げたのは、グリッドセルの横幅に対して値が大きすぎて圧迫感があったため。

## 代替案

1. **`.hero-description` の文章を短くする**: 1 行に収まるが、要件「同じ行に」を文言改変で満たすのは本質的でなく、後で文章が変わると再発する。不採用。
2. **`.status-value` の `overflow: visible` のみ追加**: ディセンダは見えるが、グラデーション文字の下端と枠の余白関係は変わらず根本解決にならない。`line-height` と `padding` を入れる方式に統一。
3. **`.status-box` 全体の `padding` を増やす**: 値ラベルの見切れは緩和されるが、ボックス自体が大きくなりすぎてカード内バランスが崩れる。要素単位の修正に留めた。
