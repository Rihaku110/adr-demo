# 0009. アプリをチャートダッシュボード化（自前 SVG 実装）

- ステータス: 採用
- 日付: 2026-05-04

## 決定

`App.jsx` をカウンタ／エラー表示／テーマ切替のデモから、サンプルデータの可視化ダッシュボードに作り変える。具体的には以下を導入する。

- 3 種類のサンプルデータセット（売上 / アクティブユーザー / コンバージョン率）と、棒グラフ / 折れ線の 2 表示形式をセグメンテッドコントロールで切替
- SVG で自作した `Chart` コンポーネント（Y 軸グリッド、X 軸ラベル、棒/折れ線/エリア、ホバー時のツールチップとクロスヘア）
- データセットから自動算出する KPI カード群（合計 / 平均 / 最大月 / 年初比成長率）
- ナビバーにテーマ切替ボタンを移設し、ブランド名を `Insight Charts` に更新
- 状態管理は `useDashboardState` カスタムフックに集約

依存ライブラリは追加せず、`<svg>` のみで実装する。

## 根拠

- 「グラフを提供するアプリ」という要件に対し、最小コストで体験価値を上げるには「データセット切替 × 表示形式切替 × 数値サマリ」の 3 点が揃ったダッシュボード形式が最も汎用的。
- Recharts / Chart.js などのライブラリ導入はバンドルが急増し、軽量デモアプリの性質と合わない。SVG であれば 12 点規模のデータでロジックがシンプル（線形スケール + パス生成）に収まり、テーマ追従（`currentColor` / グラデーション ID）も `defs` 一箇所で完結する。
- スケール計算（`niceMax`、`xCenter`、`yPos`）と Y 軸ティック生成は `useChartGeometry` に閉じ込め、`Chart` 本体は描画に専念。データセット切替時にホバーインデックスをリセットすることで、件数の異なるデータセットに切り替えた際の不整合を防ぐ。
- 旧機能（カウンタ / エラー表示）はデモ目的を果たし終えており、テーマ切替のみナビバーに移設して残した。

## 代替案

1. **Recharts や Chart.js を導入する**: 機能は豊富だが、依存追加・バンドル肥大・テーマ追従の手数（CSS 変数を反映しづらい）を考えると、12 点規模のサンプルでは過剰。将来的に多種類のチャート（ヒートマップ等）が必要になった段階で再検討する。
2. **Canvas で描画する**: パフォーマンスは出るが、アクセシビリティ（要素単位の `role`、ホバー領域）と CSS テーマ追従の両立が難しい。SVG なら `fill="currentColor"` やグラデーション ID で簡単に解決でき、データ点数も少ないため Canvas の優位性は出ない。
3. **既存のカウンタ機能を残しつつ別カードでチャートを追加**: 画面要素が増えて主従が曖昧になる。要件は「グラフを提供するアプリ」なので、ダッシュボードを主役に据えるレイアウトに振り切った。
4. **データセットを 1 つだけにする**: 実装は最小だが、「インタラクティブな可視化」の魅力が出ない。3 種類に留めることで切替の楽しさと実装の簡潔さを両立。
