# 0011. データセットを Kyndryl (NYSE: KD) サンプルに差し替え、負値対応のチャート拡張

- ステータス: 採用
- 日付: 2026-05-04

## 決定

ダッシュボードのデモデータを Kyndryl Holdings（NYSE: KD）の「月次株価終値 / 四半期売上高 / 四半期営業損益」3 種類に差し替える。あわせて以下を実施する。

- ブランド名・ヒーロー文言・フッターを `Kyndryl Dashboard` に変更（公開デモであることが伝わるよう「サンプル用の近似値」明記）
- カードのサブタイトルを各データセットの `note`（例: `NYSE: KD ※サンプル用の近似データ`）から動的に表示
- `useChartGeometry` を負値対応に拡張: `niceMin`/`niceMax`/`yZero` を導入し、Y 軸を `[niceMin, niceMax]` で線形マップ。`yPos(0)` をゼロライン基準として返す
- 棒グラフはゼロラインを起点に上方向／下方向どちらにも伸ばし、負の棒は `barGradNeg`（赤系グラデーション）で塗り分け
- 折れ線のエリア塗りも `yZero` を基準に閉じる
- 既定データセットを `stock` に変更

実値はサンプルとして近似値を使い、コード内・UI 上ともにその旨を注記する。

## 根拠

- 「Kyndryl の株価・売上データ」の要件を満たすには、(1) リアル感のある銘柄／指標と (2) 一桁オーダーが大きく異なる 3 系列を切り替えても破綻しないチャート、の両方が必要。データ自体は外部 API ではなく静的な近似値で十分（デモ用途・ネットワーク依存を避けるため）。
- 既存の `useChartGeometry` は `niceMax = ceil(maxY/5)*5` の正値前提で、営業損益のような負値を含む系列を渡すと棒の `height` が負になり描画が崩れる。`niceMin/niceMax/yZero` を導入することで、株価（正のみ）・売上（正のみ・大きな値）・営業損益（負と正が混在）の 3 系列を同じロジックで扱える。
- ゼロラインは `yPos(0)` で計算し、棒は `Math.min(yPos(d.y), yZero)` を上端、`abs(yPos(d.y) - yZero)` を高さとする標準的な双方向バー実装に揃えた。負値だけ色を変えることで、ROI 改善のトレンドが視覚的に伝わる。
- データの由来と粒度は `dataset.note` に持たせ、UI 側は単に表示するだけに留めた。今後データセットを追加する際の拡張点が 1 か所に集約される。

## 代替案

1. **Yahoo Finance / Alpha Vantage 等の外部 API から実データを取得**: リアルだが、API キー管理・CORS・レート制限・オフライン動作不可といった課題が大きく、デモ目的には過剰。`note` で「サンプル」と明記する方式に留めた。将来本番運用するなら、サーバ側プロキシ＋キャッシュを介した取得層を追加する。
2. **負値対応せず営業損益データを除外**: 実装は最も簡単だが、Kyndryl の業績トレンド（赤字 → 黒字化）こそが見せ場であり要件の趣旨を外す。チャート側を拡張して取り込む方針を選択。
3. **棒グラフの負値を `transform: scaleY(-1)` で反転表現**: 実装は短いが、ホバー領域・ツールチップ位置の補正が複雑になり、`yPos`/`yZero` を持つ素直な座標計算の方が後の保守性が高い。
4. **データセットを 1 系列（株価のみ）に絞る**: 簡潔だが、ダッシュボードとしての魅力（複数指標の切替）が損なわれる。3 系列維持。
