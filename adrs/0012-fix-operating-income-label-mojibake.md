# 0012. 営業損益ラベルの文字化け（簡体字混入）を修正

- ステータス: 採用
- 日付: 2026-05-04

## 決定

`App.jsx` の `DATASETS.operating.label` に混入していた簡体字 `损` を、日本語の正しい字体 `損` に修正する（`四半期営業损益` → `四半期営業損益`）。

## 根拠

- 直前の編集で文字列を流し込んだ際、簡体字の `损` がそのまま入ってしまっており、日本語フォントでも字形が異なるため UI 上で「文字化けした損」のように見えていた。`損` (U+640D) と `损` (U+635F) は別コードポイントの別字。
- 1 文字の置換のみで再発しないシンプルな修正。

## 代替案

1. **文字列をそのまま放置**: 不採用。日本語ユーザーには「損」の文字化けに見える。
2. **「赤字/黒字」など別表現に書き換える**: 文字化けは消えるが、他の系列ラベル命名規則（"四半期売上高" など）と不整合になる。表記の統一を優先し、字の修正に留めた。
