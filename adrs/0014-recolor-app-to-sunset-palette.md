# 0014. アプリのカラーパレットをサンセット系に変更

- ステータス: 採用
- 日付: 2026-05-08

## 決定

`change-color` スキルの呼び出しに従い、アプリ全体の配色を従来の「シアン × バイオレット × ピンク」から、ランダムに選んだサンセット系「アンバー (#fb923c) × レッド (#ef4444) × パープル (#a855f7)」に変更する。

具体的な反映箇所:

- `App.css` の CSS 変数 `--accent-cyan` / `--accent-violet` / `--accent-pink` をそれぞれ amber / red / purple に再割当
- 背景ネオン光源（`.app::before`）と `.app.dark` のラジアルグラデーションを赤紫系トーンへ
- ヒーロータイトル `.hero-title`、KPI 値 `.stat-value` のグラデーションを 3 色に追従
- SVG チャートのグラデーション ID（`barGrad` / `barGradNeg` / `areaGrad` / `lineGrad`）を新パレットへ。負値バーは新配色との対比のため青紫系に切替
- ライト時の背景は `#fff7ed → #fef2f2`（ピーチ→ローズ）、ダーク時は `#1a0e1f` 基調へ

## 根拠

- スキル定義は「ランダムな色に変更」なので、既存パレット（クール系）から最も離れた暖色寄りの 3 色グラデーションを選択することで、変化が一目で分かる。
- 既に CSS 変数 (`--accent-*`) と SVG `<defs>` のグラデーション ID にスタイルを集中していたため、配色定義の差し替えだけで全画面に伝播させられる。コンポーネントロジックや DOM 構造は触らず、最小差分で済む。
- 「上昇 = 緑 / 下落 = 赤」のセマンティックカラーはチャートのテーマと衝突しやすいため、`--up` / `--down` も暖色基調に再定義（上昇 = オレンジ、下落 = インディゴ）。色覚配慮として、▲▼ 記号と数値符号を併記する設計は維持しているため、色だけに依存せず判別可能。

## 代替案

1. **`--accent-*` 変数だけ差し替えてグラデーション直書きはそのまま**: SVG の `<defs>` や hero/stat-value のグラデーションが旧配色のままになり、画面の主役（タイトル・チャート）が変わらない。要件「色を変える」に対し効果が弱く不採用。
2. **モノクロ化やパステル化**: 「カッコよさ」を優先してきた直前の改善 (ADR 0013) のトーンを大きく崩すため不採用。
3. **テーマ切替ピッカーを実装してユーザー選択式にする**: 機能としては魅力的だが、今回の要求はワンショットの色変更であり、機能追加の範囲が広すぎる。将来的な拡張として温存。
