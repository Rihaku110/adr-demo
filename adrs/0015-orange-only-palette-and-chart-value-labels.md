# 0015. オレンジ単色基調へ統一し、チャートに値ラベルを追加

- ステータス: 採用
- 日付: 2026-05-08

## 決定

アプリ全体の配色を amber/orange/deep-orange の 3 トーン（`#fbbf24` → `#f97316` → `#c2410c`）で統一し、チャートの読みやすさを以下の方法で改善する。

- CSS 変数 `--accent-cyan/violet/pink` をすべてオレンジ系に再割当（既存スタイル側は変更最小）
- 背景ネオン光源、`.app.dark` のラジアル / 線形グラデーション、ヒーロータイトル、KPI 値、SVG `<defs>` の `barGrad / barGradNeg / areaGrad / lineGrad` を全てオレンジ階調に
- `Chart` の各データ点に `value-label` を SVG `<text>` で描画（棒は上端、負値は下端、折れ線はドット上）
- 軸ラベルのフォントを `10.5px → 12px / 500` に拡大、不透明度を 0.55 → 0.75 に引き上げ
- 値ラベルは `paint-order: stroke` + 背景色のストロークで縁取りし、グリッド・ライン上でも視認できるようにする

## 根拠

- 「オレンジ基調」要件に対し、CSS 変数とグラデーション ID にスタイルを集中していたため、配色定義の差し替えだけで全画面（背景・ヒーロー・KPI・チャート）に伝播させられる。
- 「月次株価終値が見やすく」要件に対しては、(a) 軸文字の拡大、(b) 値ラベルの常時表示、(c) ラベルの縁取り（グリッド/グラデーションエリアと衝突しない）の 3 点を併用するのが、ユーザーがホバーせずとも値を読み取れる最短距離。
- 値ラベルは `pointerEvents="none"` を付け、既存のホバー領域（バー / ドット）とのインタラクションを阻害しない構造にした。
- 紫など反対色を残すと「全体をオレンジ基調」を満たさないため、負値バーも `#b45309 → #78350f` のダークオレンジ階調に置き換え、上昇 / 下降の視認性は色の濃淡と▲▼記号で担保する。

## 代替案

1. **値ラベルを表示せず、Y 軸ティックを増やすだけ**: シンプルだが、`815 → 770 → 890` のような微差を読み取るにはホバーが必須になり「見やすさ」要件を満たしづらい。
2. **値ラベルをホバー時のみ表示**: チャートの密度は保てるが、要件「見やすく」に対し常時表示の方がストレートに応える。データ点が 12 個と少ないため常時表示でも過密にならない。
3. **配色をオレンジ単色グラデーション（同一色相のみ）にする**: トーンが単調になりカードの上端アクセントラインや KPI 値のグラデーションが弱くなる。`amber-400 → orange-500 → orange-700` の 3 段階に留め、立体感を維持した。
4. **株価チャートだけ別配色を残す**: 「全体をオレンジ基調」要件と矛盾。データセット切替時に色が変わるのは UX としても一貫性に欠けるため不採用。
