# 0016. カードヘッダーに最新値ヘッドラインを追加し、チャート余白を拡張

- ステータス: 採用
- 日付: 2026-05-08

## 決定

`card-header` 右側に `HeaderQuote` コンポーネントを追加し、現在選択中のデータセットの「最新値（大きなオレンジグラデーション）」と「前期比デルタ（▲/▼ + 数値 + %）」をピル状に表示する。あわせて `Chart` の `PADDING` を `top:20/left:48` → `top:28/left:64` に拡張し、Y 軸数値と値ラベルの可読性を上げる。

## 根拠

- 直前のリクエストで「月次株価終値が見やすく」と再度依頼があったため、すでに反映済みの値ラベル化に加えて「ユーザーが最初に視線を送るカード上部」に主要数値を据えるのが、可視性向上として最も効果が大きい。金融ダッシュボードの典型レイアウト（ヘッダー = 最新値、本体 = チャート、下段 = KPI）に合わせた。
- `PADDING.top` / `PADDING.left` の不足により、株価のような大きな数値（1,180 など）を Y 軸に出すと値ラベルがチャート上端で切れ気味だった。余白拡張で常時表示の値ラベルが見切れずに済む。
- 既存の `StatGrid` とは別の表示として `HeaderQuote` を独立させたのは、一目で目に入るべき値（最新終値）と、サマリ用 KPI（平均/ピーク/期間変化）の役割を分離するため。同じ値の重複ではあるが、視線誘導とスキャナビリティの観点から許容コスト。

## 代替案

1. **StatGrid の「最新値」カードを単に大きく表示する**: 実装は最小だが、KPI 群の中で 1 枚だけ巨大化するとレイアウトのバランスが崩れる。役割分離した方がスッキリする。
2. **チャート左上にオーバーレイで値を出す**: 動的に位置を計算する必要がありコスト高。ヘッダー固定の方が視線位置が安定する。
3. **PADDING を変えず、値ラベルだけ縮小する**: ラベルが小さくなり可読性要件に逆行するため不採用。
