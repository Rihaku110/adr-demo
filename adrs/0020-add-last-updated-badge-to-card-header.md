# 0020. カードヘッダーに「最終更新」バッジを追加する

## ステータス
Accepted

## コンテキスト
ダッシュボードはサンプルデータを表示しているが、利用者は「今表示されている数値がいつ時点のものか」を一目で把握できなかった。
ナビゲーションバーには `LIVE` バッジがあるが、これはサービス全体の稼働状態を示すものであり、特定データセットの鮮度を示していない。
データセット切替（株価／売上／営業損益）で時間軸の粒度が異なる（月次・四半期）ため、固定文言ではなく実データから動的に表示する必要があった。

## 決定
カードヘッダーの `dataset.label` / `dataset.note` の直下に、データセット内の最新期間（`data[last].x`）を表示する小さなバッジ `updated-badge` を追加する。
スタイルは既存のオレンジ系アクセントカラーと `pulse` アニメーションを再利用し、視覚的ノイズを最小化する。

## 根拠
- 「データの鮮度」というユーザーの認知ギャップを、最小の追加要素で解消できる。
- 既存トークン（アクセントカラー・mono フォント・pulse アニメーション）を流用するため、デザイン体系を崩さない。
- 表示値はデータから派生するため、データセット切替で自動追従し、メンテナンスコストがかからない。

## 代替案
1. **Hero セクションに「Last updated」を表示する**
   → グローバル情報に見えてしまい、データセット単位の鮮度を伝えにくいため不採用。
2. **`card-sub`（注記テキスト）に文字列として追記する**
   → 視覚的に強調されず、ユーザーが見逃しやすいため不採用。
3. **NavBar の LIVE バッジに統合する**
   → LIVE はサービス稼働状態を示す既存意味づけがあり、責務が混在するため不採用。

## 影響
- `react-app/src/App.jsx`: `card-header` 内に `updated-badge` 要素を追加。
- `react-app/src/App.css`: `.updated-badge` / `.updated-dot` のスタイルを追加（既存 `pulse` アニメーションを再利用）。
- 振る舞いの変更: 表示のみ。データ取得・状態管理ロジックには変更なし。
