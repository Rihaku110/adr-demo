# 0020 アプリのビジュアルを軽量化し可読性を向上する

## 決定（Decision）

ダッシュボード UI から以下の重装飾を削除・抑制し、フラットで軽量な見た目に変更する。

- `.app::before` の巨大なネオングロー（複数の radial-gradient + blur(60px) + アニメーション）を削除
- `.app::after` の全面グリッドノイズを削除
- ライト／ダークモードの背景を多重グラデーションから単色（`#fffaf3` / `#14100b`）へ変更
- `.card` の `backdrop-filter: blur` と多段ボックスシャドウを削除し、細枠＋薄い影に変更
- `.navbar` の `backdrop-filter: blur` を削除し、ほぼ不透明な単色背景に変更
- `.hero-title` と `.header-price` のグラデーションテキスト（`background-clip: text` + `text-shadow` 80px）を単色 (`#c2410c` / `#fbbf24`) に置換、hero タイトルのフォントサイズを縮小
- 折れ線チャートの `filter="url(#glow)"` (feGaussianBlur) を削除

## 根拠（Rationale）

- ユーザー要望「アプリを軽く見やすくしてほしい」に対応する。
- `filter: blur` / `backdrop-filter` / `feGaussianBlur` / グラデーション文字 + `text-shadow` は GPU 合成・再描画コストが高く、特にスクロール時とアニメーション(`floatBg`)で重くなりやすい。これらを除去することで描画負荷を大幅に下げられる。
- グラデーションテキストや巨大なグロー、グリッドノイズはコントラストを下げて文字の輪郭を曖昧にしていた。単色＋十分なコントラストにすることで数値と見出しの可読性が向上する。
- 既存のオレンジ系アクセントカラー（ADR-0015）と KPI／チャート構造（ADR-0009 系）は維持し、見た目のテーマは継承する。

## 代替案（Alternatives）

1. **ライトモード既定化のみで対応**: 重装飾は残したまま既定テーマを light に切り替える案。描画負荷は下がらず、グラデーション文字の見にくさも解消しないため不採用。
2. **完全リデザイン（別パレット・別レイアウト）**: 「見やすく」する効果は大きいが、ADR-0013〜0016 で積み上げてきたデザイン方針を大きく覆すことになり、要求（"軽く"）に対して過剰なため不採用。
3. **CSS の `prefers-reduced-motion` 対応のみ**: アニメーションは抑えられるが、blur や多重シャドウなどの常時コストは残るため不採用。
