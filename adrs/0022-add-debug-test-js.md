# 0022. デバッグ用 test.js を追加する

## 決定

`react-app/src/test.js` を新規作成し、`console.log("hook test")` を1行記載する。

## 根拠

- 動作確認やフック検証用に、最小構成の独立したスクリプトが必要だった。
- 既存の `App.jsx` や `main.jsx` を改変せずに検証できるよう、専用ファイルとして分離した方が影響範囲を限定できる。
- 1行のログ出力に留めることで、検証以外の副作用を生まない。

## 代替案

- **App.jsx に直接 console.log を追加**: 既存コンポーネントへ検証用コードを混入させると、削除漏れや本番影響のリスクがあるため不採用。
- **テストフレームワーク (Vitest 等) でテストケース化**: 単発の動作確認には過剰であり、まずは軽量なスクリプトで足りるため不採用。
- **ファイルを作らず手動で console を実行**: 再現性が無く、フック等の検証ポイントを残せないため不採用。
